## Format Changelog

### Dla Przyszłych Zmian

Każde wydanie powinno zawierać:

```markdown
## [X.Y.Z] - YYYY-MM-DD

### 🎉 Dodane
- Nowa funkcja 1
- Nowa funkcja 2

### 🔧 Zmienione
- Zmiana 1
- Zmiana 2

### 🐛 Naprawione
- Bug 1
- Bug 2

### ❌ Usunięte
- Stara funkcja
```

### Wersjonowanie

Używamy [Semantic Versioning](https://semver.org/):
- **MAJOR** (.0.0) - Breaking changes
- **MINOR** (0.0.0) - Nowe funkcje
- **PATCH** (0.0.0) - Bug fixes
